# 脅威モデリング ガイド

「どんなリスクがあるか整理して」「脅威を洗い出したい」という要求に使う。
アプリケーションの性質に応じて、攻撃面・影響・対策の優先度を提示する。

---

## フロントエンド脅威マップ（STRIDE対応）

| 脅威カテゴリ | フロントでの具体的脅威 | 主な対策 |
|---|---|---|
| **Spoofing** (なりすまし) | セッション固定攻撃、クッキー盗難 | HttpOnly/Secure Cookie, CSRF Token |
| **Tampering** (改ざん) | DOM改ざん(XSS)、リクエスト改ざん | CSP, SRI, バリデーション |
| **Repudiation** (否認) | ユーザー操作の証跡なし | 監査ログ(サーバー側) |
| **Information Disclosure** (情報漏洩) | ソースマップ、環境変数露出 | ソースマップ非公開, env分離 |
| **Denial of Service** | 過負荷リクエスト | Rate limiting(サーバー側) |
| **Elevation of Privilege** | 権限昇格(水平・垂直) | サーバー側の認可、最小権限 |

## ユーザータイプ別の攻撃シナリオ

### 1. 外部の攻撃者（認証なし）
- **シナリオA: 反射型XSS**
  - URLにスクリプトを埋め込んだリンクをユーザーに踏ませる
  - 影響: セッション盗難、フィッシング
  - 対策: CSP, 入力サニタイズ, HTTPOnly Cookie

- **シナリオB: クリックジャッキング**
  - iframe内に正規サイトを埋め込み、ユーザーにクリックさせる
  - 影響: 意図しない操作（送金・削除など）
  - 対策: X-Frame-Options, CSP frame-ancestors

- **シナリオC: サプライチェーン攻撃**
  - CDNやnpmパッケージを汚染してマルウェアを注入
  - 影響: 全ユーザーへの影響
  - 対策: SRI, npm audit, lock file

### 2. 認証済み悪意あるユーザー（内部脅威）
- **シナリオD: Stored XSS**
  - プロフィールやコメントにスクリプトを保存 → 他ユーザーが閲覧時に実行
  - 影響: 他ユーザーのセッション盗難
  - 対策: サーバー側サニタイズ, CSP, DOMPurify

- **シナリオE: CSRF**
  - 罠サイトから認証済みユーザーの権限でリクエストを実行させる
  - 影響: データ変更・削除、送金
  - 対策: CSRF Token, SameSite Cookie

### 3. 開発・インフラリスク
- **シナリオF: 機密情報の意図しない公開**
  - APIキー・DBパスワードをクライアントバンドルに含める
  - 影響: バックエンドへの不正アクセス
  - 対策: 環境変数の分離, コードレビュー

## アプリタイプ別リスク優先度

### ECサイト・金融系（高価値ターゲット）
優先順位: CSRF > XSS > セッション管理 > 機密情報露出 > CORS

### 管理画面・ダッシュボード
優先順位: XSS(Stored) > CSRF > 認証強化 > CSP > 権限管理

### ブログ・コンテンツサイト
優先順位: XSS > セキュリティヘッダー > SRI > サードパーティリスク

### SPA（Single Page Application）
優先順位: JWT/Token管理 > XSS > CORS > CSP > ソースマップ
