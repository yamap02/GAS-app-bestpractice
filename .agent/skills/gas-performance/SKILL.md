# GAS Webアプリ パフォーマンスリファクタリングスキル

GAS（Google Apps Script）のWebアプリ特有の制約を踏まえ、
クライアント・サーバー両面からボトルネックを特定・修正する。

---

## GAS特有の制約（必ず念頭に置くこと）

| 制約 | 内容 | 影響 |
|---|---|---|
| **シングルスレッド（クライアント）** | ブラウザのJSはシングルスレッド。`google.script.run`の呼び出し中もUIスレッドは動く | コールバック設計が必須。ブロッキング処理でUIがフリーズ |
| **サービスワーカー不在** | GAS WebアプリはService Workerを登録できない | オフラインキャッシュ・バックグラウンド同期が使えない。代替戦略が必要 |
| **サーバー呼び出しコスト** | `google.script.run`は1回あたり数百ms〜数秒 | N+1呼び出しは致命的。バッチ化が最重要 |
| **実行時間制限** | サーバー側スクリプトは最大6分（コンシューマ）/ 30分（Workspace） | 長時間処理はチャンク分割＋PropertiesServiceで状態管理 |
| **同時実行制限** | ユーザーごとに同時実行数に上限あり | LockServiceで排他制御が必要な場面も |
| **ScriptApp quota** | URLFetch・メール送信・スプレッドシート操作に1日の上限 | 無駄な呼び出しを減らすことがそのままコスト削減に直結 |

## スキル実行フロー

### Step 1: コンテキスト把握

以下をユーザーの入力から読み取る（不明な場合は質問する）：

- **処理の種類**: スプレッドシート読み書き / Drive操作 / 外部API / メール送信
- **データ規模**: 行数・セル数・ファイル数の目安
- **UIの有無**: HTMLサービス使用 / サイドバー / モーダル / スタンドアロンWebアプリ
- **現在の問題**: タイムアウト / UI固まり / 応答が遅い / 特定操作だけ遅い
- **コードの提供**: あればレビューモードへ、なければ質問で絞り込む

### Step 2: 適切な参照ファイルを読み込む

モードに応じて `references/` 以下を `view` ツールで読み込む。
複数モードにまたがる場合は複数ファイルを読む。

### Step 3: ボトルネック特定 → 修正案提示

以下の優先順位で問題を整理して提示する：

```
🔴 Critical  - タイムアウト・完全フリーズ・quota超過
🟠 High      - 体感できる遅延（>2秒）・N+1呼び出し
🟡 Medium    - 改善余地あり（500ms〜2秒）・コードの可読性リスク
🟢 Low       - 軽微な最適化・将来的なスケール懸念
```

## クイック診断チェック

コードを見たらまずこれを確認する：

### クライアント側（HTML/JS）
- [ ] `google.script.run` をループ内で呼んでいないか → **N+1問題**
- [ ] 呼び出し結果を待ってからUIを更新しているか → **ブロッキングUI**
- [ ] 大量のDOM操作を個別に行っていないか → **DocumentFragment未使用**
- [ ] イベントリスナーを使い回せているか → **重複バインド**
- [ ] データを毎回サーバーから取得していないか → **クライアントキャッシュ不足**

### サーバー側（.gs）
- [ ] セルを1つずつ `getValue()` / `setValue()` していないか → **バッチ化必須**
- [ ] ループ内でSpreadsheet/Drive/URLFetchを呼んでいないか → **呼び出し最小化**
- [ ] `CacheService` を活用しているか → **キャッシュ未利用**
- [ ] 巨大なデータをそのままクライアントに返していないか → **ペイロード最適化**
- [ ] LockServiceなしで競合が起きうる処理か → **排他制御漏れ**

## 出力スタイルガイドライン

- **コードレビュー**: 問題箇所（Before） → 問題の説明 → 修正コード（After） の順で提示
- **重要度**: 🔴/🟠/🟡/🟢 で明示
- **数値根拠**: 「1回のgetValue()は約50ms」など具体的なコスト感を添える
- **代替手段**: Service Workerが使えない場面では `CacheService` / `PropertiesService` / クライアントの `localStorage` などの代替を提示
- 日本語で回答、コードはそのまま英語（コメントも英語OK）
- GASの公式ドキュメントURLは必要に応じて末尾に添付
