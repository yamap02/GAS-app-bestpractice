# GAS Webアプリ パフォーマンス チェックリスト

コードレビューや自己点検に使える項目リスト。重要度付き。

---

## 🔴 Critical（まず確認する）

### クライアント側
- [ ] **N+1呼び出しなし**: `google.script.run` をループ内で呼んでいない
- [ ] **非同期UI**: サーバー処理中にローディング表示があり、UIがフリーズしない
- [ ] **withFailureHandler設定**: 全ての `google.script.run` にエラーハンドラがある

### サーバー側
- [ ] **一括読み書き**: `getValue()`/`setValue()` の代わりに `getValues()`/`setValues()` を使用
- [ ] **ループ内APIなし**: ループ内でSpreadsheet/Drive/URLFetchを呼んでいない
- [ ] **タイムアウト対策**: 大量データ処理はチャンク分割 + PropertiesService で再開可能

---

## 🟠 High（次に確認する）

### クライアント側
- [ ] **ペイロード最適化**: 必要なデータだけ取得（全カラム丸ごと送信していない）
- [ ] **DOM操作最小化**: テーブル描画に `DocumentFragment` を使用
- [ ] **イベント委譲**: 動的要素のイベントは親要素にバインド

### サーバー側
- [ ] **CacheService活用**: 頻繁に変わらないデータ（マスターデータ等）をキャッシュ
- [ ] **LockService**: 複数ユーザーが同時操作しうる書き込み処理に排他制御あり
- [ ] **SpreadsheetApp.flush()**: 書き込み後に明示的にflushしている（必要な場合）
- [ ] **fetchAll使用**: 複数の外部API呼び出しは `UrlFetchApp.fetchAll()` で並列化

---

## 🟡 Medium（余裕があれば確認する）

### クライアント側
- [ ] **クライアントキャッシュ**: ドロップダウン等の静的データをメモリキャッシュ
- [ ] **ページネーション**: 大量データは一括取得せず、ページ単位 or 無限スクロール
- [ ] **デバウンス**: 検索・フィルタ入力に debounce をかけている（300ms程度）
- [ ] **仮想スクロール**: 1000行超のテーブルは仮想スクロール検討

### サーバー側
- [ ] **不要なgetActive()なし**: `SpreadsheetApp.getActive()` を毎回呼ばず変数にキャッシュ
- [ ] **Logger.log削減**: 本番コードに大量の `Logger.log` / `console.log` がない
- [ ] **範囲の最小化**: `getDataRange()` ではなく必要な範囲だけ `getRange()` で取得

---

## 🟢 Low（スケールを見据えて）

- [ ] **キャッシュTTLの設計**: データの更新頻度に合わせたTTLを設定
- [ ] **エラーリトライ**: 一時的な失敗に対するリトライロジックがある
- [ ] **Quotaモニタリング**: 1日のAPI呼び出し数が quota の80%以下
- [ ] **トリガー管理**: 不要な時間ベーストリガーが残っていない

---

## サービスワーカー代替チェックリスト

GAS WebアプリはService Workerが使えないため、以下の代替手段で補う：

| Service Workerの機能 | GASでの代替手段 |
|---|---|
| レスポンスキャッシュ | `CacheService`（サーバー側）/ メモリキャッシュ（クライアント側） |
| オフライン対応 | `localStorage` への書き込み + オンライン時に同期 |
| バックグラウンド同期 | 時間ベーストリガー + `PropertiesService` で状態管理 |
| プッシュ通知 | `MailApp` / `GmailApp` でのメール通知（代替） |
| プリキャッシュ | 初回ロード時に必要データをまとめて取得しておく |

## スコアリング目安

| スコア | 状態 | アクション |
|---|---|---|
| Critical 全✅ | 最低限OK | Highの修正を進める |
| Critical に❌あり | 要緊急対応 | まずCriticalを全て修正 |
| High 全✅ | 良好 | Mediumを改善余地として記録 |
| Medium・Low | 任意 | スケールや要件に応じて対応 |
